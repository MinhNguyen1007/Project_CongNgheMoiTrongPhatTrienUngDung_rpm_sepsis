## 8.6. Nguyên tắc thiết kế

- **Ưu tiên thông tin quan trọng nhất ở vị trí nổi bật**: xác suất sepsis + mức độ nghiêm trọng luôn hiện rõ nhất.
- **Màu sắc có ngữ nghĩa lâm sàng**:
  - 🔴 Đỏ: nguy cơ nghiêm trọng (≥ 70%)
  - 🟡 Vàng: cảnh báo (50-69%)
  - 🟢 Xanh: an toàn (< 50%)
  - Không dùng màu khác cho nguy hiểm (tránh gây nhầm lẫn).
- **Giảm thiểu mệt mỏi do cảnh báo**: một bệnh nhân đang có cảnh báo hoạt động không tạo thêm cảnh báo trong 1 giờ.
- **Luôn có ghi chú cho mỗi quyết định lâm sàng**: phục vụ audit và pháp lý.
- **Thông tin bối cảnh (context)**: hiển thị SHAP top features để bác sĩ hiểu lý do mô hình đưa cảnh báo.
- **Responsive**: tối thiểu hỗ trợ tablet (trên ICU dùng iPad là phổ biến).
- **Tuân thủ WCAG 2.1 AA**: contrast ratio ≥ 4.5:1, hỗ trợ keyboard navigation.

## 8.7. Stack UI

- **Framework**: React 18 + TypeScript + Vite
- **Styling**: TailwindCSS + shadcn/ui (thành phần cơ bản)
- **Charts**: Recharts (biểu đồ sinh hiệu, xác suất)
- **State**: Zustand cho UI state, TanStack Query cho server state
- **Realtime**: native WebSocket + custom hook `useWebSocket`
- **Icons**: Lucide React
- **Font**: Inter (hệ thống) + JetBrains Mono (số sinh hiệu)
